# Development Rules

The following rules apply to all development activities in this repository:

1. **Branching Strategy**: 
   - **Never develop on the `main` branch.**
   - Always create a dedicated feature branch for new features, bug fixes, or chores (e.g., `feature/add-database`, `bugfix/fix-checkout-bug`).

2. **Commit Policy**:
   - Committing directly to `main` is strictly prohibited.
   - Merging or committing to the `main` branch is only permitted upon explicit user confirmation.
